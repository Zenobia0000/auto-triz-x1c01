# AICBD 工業 CV 領域：核心底層觀念與北極星指標

本文件從 AICBD 職務使命出發，歸納工業電腦視覺 AI 系統架構師在製造現場必須掌握的底層觀念，以及該關注的北極星指標體系。

---

## 一、AICBD 三大使命

| 使命 | 描述 |
| :--- | :--- |
| 提升組裝直通率 | AI 平台蒐集分析工站影像，減少錯漏裝與重工 |
| 提高產品品質 | 關鍵步驟導入 AI 視覺檢測，穩定出貨品質 |
| 縮短新產品導入時間 | AI 技術加速新品導入流程 |

---

## 二、5 個核心底層觀念

### 1. Yield-Driven Thinking（良率驅動思維）

製造業 AI 的存在意義只有一個：**讓良率上升、重工下降**。所有技術決策都必須回答：「這件事做完，直通率會提升幾個百分點？」

- **漏檢（Escape）** 比 **誤報（False Alarm）** 嚴重一百倍——漏檢 = 不良品出貨 = 客訴 = 賠錢
- **誤報太多** = 現場停線複檢 = 產能損失 = 沒人信任系統
- 兩者的 trade-off 不是數學問題，是**商業決策**——不同客戶、不同產品線的容忍度完全不同

> 核心：AI 模型的精度不是目標，良率改善才是目標。模型只是工具。

---

### 2. Takt Time Constraint（節拍時間約束）

工廠產線有節拍時間（takt time），每個工站的處理時間是硬約束。CV 推理必須在這個時間窗口內完成：

- 推理太慢 → 擋住產線 → 比沒有 AI 還糟
- 批次離線檢 → 延遲發現不良 → 重工成本暴增

意味著：
- **模型大小、推理延遲、硬體選型**不是事後優化，是架構設計的第一約束
- Edge inference vs. server inference 的取捨是架構決策
- 模型壓縮（pruning、quantization、distillation）不是加分項，是必備能力

> 核心：產線不會等你的模型跑完。延遲是硬天花板，不是可調參數。

---

### 3. Domain Shift is the Norm, Not the Exception（場域漂移是常態）

製造現場不是固定的實驗室：

- **光源變化**：白班 vs. 夜班、燈管老化、季節日照角度
- **物料批次差異**：同料號不同供應商，外觀微妙不同
- **治具磨損**：夾具老化導致定位偏移
- **產品換線**：今天做 A 機種，明天做 B 機種，後天做全新機種
- **作業員差異**：每個人放料的角度、力道不同

意味著：
- **模型上線第一天的準確率是最高點，之後只會退化**
- 必須有 retraining pipeline、drift detection、持續標記機制
- 「一次訓練永久使用」在工業 CV 不存在

> 核心：你部署的不是模型，是一個持續適應環境變化的系統。

---

### 4. Labeling Cost is the Real Bottleneck（標記成本是真正的瓶頸）

在製造 CV 領域：

- 標記需要**領域專家**（品保工程師），不是隨便找人就能標
- 不良品本身就是**長尾分佈**——良品 99%+，不良品稀少且樣態多變
- 每個新機種 = 幾乎從零開始標記
- 標記品質直接決定模型天花板

意味著：
- **資料增強（augmentation）、合成資料（synthetic data）、few-shot learning** 不是花招，是生存手段
- **主動學習**：讓模型告訴你「哪張圖標了收益最大」，而非盲標
- **Pre-labeling + 人工修正**的流程設計決定標記效率能提升幾倍
- **標記一致性（inter-annotator agreement）**比標記數量重要

> 核心：資料不是免費的。每一張標記都有成本，系統設計必須最小化標記需求、最大化每張標記的價值。

---

### 5. Closed-Loop Feedback（閉環回饋機制）

工業 CV 不是「部署完就結束」，是一個閉環：

```
影像採集 → 模型推理 → 判定（OK/NG）→ 人工複檢 → 回饋標記 → 模型更新 → ...
```

如果這個環斷掉任何一段：
- 沒有人工複檢 → 不知道模型錯在哪
- 複檢結果沒有回流 → 模型永遠不會進步
- 沒有版本管理 → 不知道哪個模型在哪條線、效果如何

> 核心：系統的價值不在第一次部署的準確率，而在它能以多快的速度自我修正。

## 三、北極星指標體系

### 3.1 業務層北極星（直接對應 AICBD 三大目標）

| 指標 | 定義 | 為什麼是北極星 |
|:-----|:-----|:---------------|
| **First Pass Yield (FPY)** | 產品第一次通過所有工站檢測的比率 | AICBD 存在的終極理由。AI 的每一分投入都要能回答「FPY 提升了多少」 |
| **NPI Cycle Time** | 新產品從立項到量產的時間 | 直接衡量「AI 有沒有加速新品導入」。含模型開發、標記、驗證、上線全流程 |
| **Outgoing Quality Level (OQL)** | 出貨後客訴 / 退貨率 | 漏檢的終極後果。這個數字惡化 = AI 檢測系統失敗 |

### 3.2 系統層指標（架構師的儀表板）

| 指標 | 定義 | 閾值邏輯 |
|:-----|:-----|:---------|
| **Escape Rate（漏檢率）** | 真實不良品被判為 OK 的比率 | 最致命指標。通常 < 0.1% |
| **False Call Rate（誤報率）** | 良品被判為 NG 的比率 | 太高 → 現場關掉系統。通常需 < 1-2% |
| **Inference Latency（推理延遲）** | 單張影像從輸入到判定的時間 | 必須 < takt time |
| **Model Drift Score** | 模型在線準確率相對上線時的衰退幅度 | 觸發 retrain 的關鍵信號 |
| **Labeling Efficiency** | 每張可用標記的平均耗時（含 pre-label + 修正） | 衡量人機標記協作流程的效率 |
| **Time-to-Deploy（新機種上線時間）** | 新產品 CV 模型從需求到上線的天數 | 直接影響 NPI |

### 3.3 營運層指標（每日/每週追蹤）

| 指標 | 定義 | 用途 |
|:-----|:-----|:-----|
| **Daily Inspection Volume** | 每日檢測量 | 系統穩定性的基本脈搏 |
| **Human Override Rate** | 人工覆寫 AI 判定的比率 | 過高 = 模型不可信；過低 = 可能沒人在看 |
| **Retrain Frequency** | 模型重訓頻率 | 衡量系統維護負擔 |
| **Active Learning Hit Rate** | 主動學習選出的樣本被專家確認為有價值的比率 | 衡量標記策略的精準度 |
| **Cross-Site Variance** | 同一模型在不同廠區的準確率差異 | 差異大 = domain shift 嚴重 |

## 四、指標之間的因果關係

```
Labeling Efficiency ──→ Time-to-Deploy ──→ NPI Cycle Time
        │                                        │
        ↓                                        ↓
Model Accuracy ──→ Escape Rate ──────────→ OQL (客訴率)
        │               │
        ↓               ↓
False Call Rate ──→ 現場信任度 ──→ 系統是否被真正使用
        │
        ↓
Human Override Rate（信號：模型該更新了）
        │
        ↓
Model Drift Score ──→ Retrain 觸發 ──→ 閉環
```

**最關鍵的因果鏈**：

> **Escape Rate 惡化 → OQL 下降 → 客訴 → AI 系統被質疑**
> **False Call Rate 過高 → 現場關掉系統 → AI 投入歸零**

這兩條鏈是整個系統的生死線。所有底層觀念（良率驅動、節拍約束、場域漂移、標記成本、閉環回饋）最終都是在守住這兩條線。

---

## 五、收斂

| # | 底層觀念 | 解決什麼 | 一句話 |
|:--|:---------|:---------|:-------|
| 1 | 良率驅動 | 技術脫離業務 | 先問「良率提升多少」，不是「mAP 多高」 |
| 2 | 節拍約束 | 推理擋線 | 延遲是硬天花板，不是可調參數 |
| 3 | 場域漂移是常態 | 模型退化 | 部署的是系統，不是模型 |
| 4 | 標記成本是瓶頸 | 資料不夠 | 最小化標記需求，最大化每張標記價值 |
| 5 | 閉環回饋 | 無法自我修正 | 價值在於修正速度，不在首次準確率 |

> **盯 Escape Rate 和 False Call Rate 這對矛盾——它們的平衡點不是技術問題，是商業判斷；而維持這個平衡點不漂移，就是整個 AI 系統架構存在的理由。**
