# Auto-TRIZ UML 導覽 — 新成員快速上手

## 這個專案在做什麼？

Auto-TRIZ 是一套**系統性矛盾解決框架**，將 TRIZ（發明問題解決理論）的三大工具 — 技術矛盾 (TC)、物理矛盾 (PC)、質-場分析 (SF) — 編排成可重複執行的閉環流程，用於 ebike 機電系統的設計決策。

核心思路：**TC → PC → SF 不是平行選項，而是同一問題在不同解析度下的嵌套式收斂路徑。**

| 工具 | 解析度 | 角色 | 產出 |
|:-----|:------|:-----|:-----|
| TC | 10x 鏡頭 | 外交官 | 啟發性方向 |
| PC | 100x 鏡頭 | 外科醫生 | 決定性分割策略 |
| SF | 手術刀路徑 | 配方工程師 | 具體結構方案 |

---

## 5 分鐘快速理解路線

如果時間有限，按此順序讀 3 份文件即可掌握全貌：

```
① 00_domain_model.md     — 核心概念與關係（靜態結構）
② 02_main_flow.md        — 完整閉環流程（動態行為）
③ 10_problem_lifecycle.md — 狀態機總覽（終止條件）
```

讀完這三份，你會知道：系統裡有哪些概念、它們怎麼流動、最終在哪裡結束。

## 關鍵術語速查

| 術語 | 全稱 | 一句話解釋 |
|:-----|:-----|:---------|
| **TC** | Technical Contradiction | 改善 A 導致 B 惡化 |
| **PC** | Physical Contradiction | 同一參數同時要兩種相反狀態 |
| **SF** | Su-Field (Substance-Field) | 物質 + 場的交互模型 |
| **Px** | 中介物理變數 | 同時控制 TC 兩端的底層物理量 |
| **OZ** | Operational Zone | 衝突發生的具體空間位置 |
| **OT** | Operational Time | 衝突發生的時間點 |
| **SIM** | Solution Interaction Matrix | 多 TC 解法的兩兩相容性矩陣 |
| **IFR** | Ideal Final Result | 最後理想結果：功能自行實現，零成本零危害 |

## 典型使用場景

### 場景 A：我有一個設計問題，想用 TRIZ 分析

```
01 入口判定 → 確認 Level A/B → 02 主流程 → 逐步執行
```

### 場景 B：流程卡在 Px 找不到

```
04 Px 搜索 → 4.2.2 三種情境 → 09 喊停 vs 繼續衝
```

### 場景 C：多個矛盾同時存在

```
05 多TC策略 → 瓶頸 or SIM？ → 06 SIM 迭代 → 07 迴圈診斷（如有）
```

### 場景 D：解法出來了，怎麼判斷好不好

```
08 複雜度判定 → 進化就通過 / 補丁就決策
```

## 文件間的參照關係

所有 UML 圖是策略文件 `docs/auto_triz_strategy.md` 的視覺化拆解。關係如下：

```
docs/auto_triz_strategy.md        ← 完整文字描述（元文件）
  │
  ├── docs/uml/00-10              ← 視覺化流程圖（本目錄）
  │
  └── triz_knowledge_base/        ← 靜態參照表（查表用）
      ├── 01_39_parameters.md
      ├── 02_contradiction_matrix.md
      ├── 03_40_principles.md
      ├── 04_separation_principles.md
      └── 05_76_standard_solutions.md
```

遇到 UML 圖看不懂時，回去讀策略文件對應章節的文字描述。
