# TRIZ Step 4: 驗證與複雜度檢查

載入 **triz-verify** skill，驗證解法方案並產出工程規格書或技術債記錄。

## 使用方式

```
/triz-verify                       # 從 Step 3 產出接續
```

## 驗證流程

1. **Px 分離驗證** — 解法是否真正分離了物理矛盾
2. **四問複雜度判定** — 結構/能量/認知/演化對齊
3. **補丁 vs 進化** — 決定路徑
4. **新 TC 偵測** — 檢查是否引入新矛盾
5. **下游交付物** — 工程規格書 / 驗證計畫 / 技術債記錄
