# TRIZ Step 4: 原型驗證 + 複雜度檢查

## Overview

本 skill 實作 Auto-TRIZ 閉環流程的 Step 4（Section 6 of `docs/auto_triz_strategy.md`）。

接收 Step 2-3 產出的解法方案（F/S/OZ/OT + 分離策略 + 標準解），執行：
1. Px 分離驗證
2. 四問複雜度判定（補丁 vs 進化）
3. 新 TC 偵測
4. 下游交付物產出

**宣告：** 「正在使用 triz-verify skill — 執行 Step 4: 驗證與複雜度檢查。」

## Phase 1: Px 分離驗證

### 1.1 邏輯驗證

用 PC 造句檢查解法是否真正分離了 Px：

```
原始 PC：Px 必須是 [A]（為了 P1）且 [非 A]（為了 P2）
解法後：
  - 在 [分離維度的一側]，Px = A → P1 滿足 ✓/✗
  - 在 [分離維度的另一側]，Px = 非 A → P2 滿足 ✓/✗
```

### 1.2 驗證結果判定

| 結果 | 動作 |
|:-----|:-----|
| 兩側皆滿足 | 通過，進入 Phase 2 |
| 一側不滿足 | 分離不完整 — 回到 Step 3 調整分離策略 |
| 兩側皆不滿足 | 解法無效 — 回到 Step 2 重新選擇原理 |

---

## Phase 2: 四問複雜度判定

依序問四個問題：

### Q1: 結構複雜度 (Structural)
> 「零件/組件是否變多了？」

- **是** = 新增零件修正舊零件 → 補丁信號
- **否** = 現有零件多功能化 → 進化信號

### Q2: 能量複雜度 (Energy)
> 「能量消耗是否變大了？」

- **是** = 引入新能量系統 → 補丁信號
- **否** = 利用環境資源替代 → 進化信號

### Q3: 認知複雜度 (Cognitive)
> 「別人理解這個邏輯的時間是否變長了？」

- **是** = 依賴鏈變長 → 補丁信號
- **否** = 縮短轉換步驟 → 進化信號

### Q4: 演化對齊 (Evolution Alignment)
> 「解法方向是否符合以下至少一個演化趨勢？」

- 理想度增加（更多功能，更少成本/危害）
- 朝微觀發展（材料層級解決）
- 動態化增加（感測器驅動的狀態切換）

### 判定規則

```
Q1-Q3 中有 ≥2 個「是」 → 補丁
           或
Q4 為「否」            → 補丁
           否則
                       → 進化
```

---

## Phase 3: 補丁 vs 進化路徑

### 進化路徑
解法品質良好，產出下游交付物（Phase 4）。

### 補丁路徑

```
判定為補丁
│
├─ 時間/成本允許重新深挖？
│  ├─ 是 → 建議回到 Step 3 (PC) 重新深挖
│  │       詢問使用者是否要返回
│  │
│  └─ 否（出貨壓力）→ 有意識地選擇補丁：
│     1. 記錄為「已知技術債」
│     2. 標記哪個 PC 未被真正解決
│     3. 排入下一代 TRIZ 待辦清單
│     4. 繼續產出交付物
```

---

## Phase 4: 新 TC 偵測

檢查解法是否引入新的技術矛盾：

```
新方案是否引入新的 TC？
│
├─ 否 → 完成
│
├─ 新 TC 與原 TC 形成迴圈
│  （解 A → 惡化 B → 解 B → 惡化 A）
│  └─ 進入迴圈診斷（檢查 OZ/OT/Px 重疊）
│     └─ 重疊 → 其實是同一個未正確定義的 PC
│     └─ 不重疊 → 兩個獨立 TC
│
└─ 獨立新 TC → 回 Step 1（螺旋上升）
```
